# AGENTS.md

## Project overview

このプロジェクトは、作業セッションを収集と成長に変換するWebアプリのMVPである。
画像、演出、ドット絵は後回し。
最初は骨格、状態、データ、計算ロジックを優先する。

## Read first

実装前に以下を読むこと。

1. docs/product-spec.md
2. docs/coding-standards.md
3. README.md

## Working rules

- MVP範囲外の機能を勝手に追加しない
- UIの見た目改善を優先しない
- ロジックを画面コンポーネントに直書きしない
- セッション状態は時刻ベースで管理する
- 秒数を真実の状態として保存しない
- started_at, paused_at, ended_at, pause_accumulated_seconds を基本に扱う
- 変更は最小差分で行う
- 既存ファイルの責務を壊さない
- 不要な依存を増やさない
- 命名は短さより明快さを優先する
- コメントは「なぜ」を書く
- 「何をしているか」はコードで読めるようにする

## MVP priority

最優先は以下。

1. 用途フィールド
2. セッション開始
3. 休憩
4. 再開
5. 終了
6. 結果表示
7. スコア計算
8. 経験値加算
9. 図鑑
10. 近傍ランキング

## Explicitly out of scope for now

- ドット絵
- アニメーション
- 通知
- フレンド
- チャット
- SNS共有
- 全体ランキング
- 高度な不正対策
- デスクトップアプリ化

## Architecture expectations

- UIとドメインロジックを分離する
- 計算ロジックは lib に置く
- DBアクセスは server 側に寄せる
- 画面は表示と操作に集中させる
- 将来のアプリ化を阻害しない構造を保つ

## Session rules

セッション状態は以下の遷移に限定する。

idle -> working
working -> paused
paused -> working
working -> ended
paused -> ended

それ以外の状態遷移は作らない。

## Scoring rules

- 長時間耐久を褒めない
- 45〜60分前後の集中を推奨帯とする
- 5〜10分前後の休憩を推奨帯とする
- 120分超の連続作業は効率面で不利にする
- ランキングは総時間ではなく週次スコアを使う

## Done when

タスク完了条件は以下。

- 仕様に沿っている
- 余計な機能を足していない
- 命名が明快
- 責務分離が守られている
- 変更したファイルごとの責務を説明できる
- lint が通る
- test がある場合は通る

## Output expectations

実装後は以下を簡潔に説明すること。

- 変更したファイル
- 各ファイルの責務
- なぜその構成にしたか
- 代替案があるなら何か
- 今回あえて入れていないもの
