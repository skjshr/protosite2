# コレクション（図鑑）体験の設計

## 概要

セッション終了時にアイテムがドロップし、テーマ別の図鑑として収集できる。
実装詳細は `design/item-reward-system.md` を参照。

## ユーザーの体験

- セッションを終えるたびにアイテムが手に入ることがある
- アイテムはフィールドに設定したテーマ（職業）に対応する
- 「鉱夫」フィールドで作業すれば鉱夫系のアイテムが集まる
- 長時間耐久より良いリズムで作業した方が希少アイテムが出やすい

## コレクション画面の体験

- ホーム画面の「コレクション」タブから閲覧できる
- 自分が持っているテーマのタブだけ表示される
  - miner フィールドを持っていれば「鉱夫」タブが出る
  - 持っていないテーマのタブは出ない
- 取得済みアイテムは名前・レアリティ・説明文が見える
- 未取得アイテムは「???」で表示される（存在は分かる）
- 全体の取得率（○○/△△個）が分かる

## アイテムの希少度

| レアリティ | 特徴 |
|-----------|------|
| common | よく出る。種類が多い |
| rare | 出にくい。スコアが高いほど確率が上がる |
| epic | 滅多に出ない。コレクター欲を刺激する |

## テーマとアイテムのスケール感

- テーマは今後どんどん増やしていく（将来は課金ユーザーが自作できる）
- 各テーマに数百種類のアイテムを用意していく
- シードデータで徐々に追加していく（スキーマ変更不要）

## レベルと解放の関係

- 一部のアイテムはレベルが一定以上でないとドロップ対象に入らない（`unlockLevel`）
- 低レベルのうちは出ないアイテムが、レベルアップにより解放される
- 図鑑画面では解放されていないアイテムも「???」で存在が見える（ロック状態の可視化）

## 今後の拡張イメージ

- テーマの追加（農夫・錬金術師・探検家 など）
- 課金ユーザーによるオリジナルテーマ作成
- テーマ内のアイテムをユーザーが定義できる仕組み
