# セッション体験の設計

## 概要

ユーザーが作業を開始してから終了するまでの体験を定義する。
実装詳細は `design/work-session.md` および `design/screen-components.md` を参照。

## ユーザーの体験フロー

```
① ホームにアクセス
   ↓
② フィールド（用途）を選ぶ
   例：「英語」「コーディング」「読書」
   ↓
③ 開始ボタンを押す
   → タイマーが動き始める
   ↓
④a 休憩ボタンを押す（疲れたら）
   → タイマーが止まる（休憩中）
   → 再開ボタンを押す → ③に戻る
④b 終了ボタンを押す（作業を終えたら）
   ↓
⑤ 結果が同じ画面に表示される
   - 有効時間
   - スコア（リズムの良さ）
   - 獲得XP
   - レベルアップしたか
   - 獲得アイテム（あれば）
   ↓
⑥ 次のセッションを始める or コレクションを見る
```

## セッション状態の遷移

```
idle
  ↓ 開始
working
  ↓ 休憩        ↓ 終了
paused         ended
  ↓ 再開  ↓ 終了
working  ended
```

- **それ以外の遷移は存在しない**
- `idle` はアプリ起動時の初期状態（DBには存在しない）
- `ended` になったセッションは変更できない

## スコアの直感的な意味

| 状況 | スコアへの影響 |
|------|--------------|
| 10分未満で終了 | スコア 0（カウントされない） |
| 45〜60分の集中 | 最高倍率（推奨帯） |
| 5〜10分の休憩 | 休憩ボーナス |
| 120分超の連続作業 | 倍率ダウン（長時間耐久を褒めない） |

スコアは「どれだけ長く作業したか」ではなく「どれだけ良いリズムで作業したか」を表す。

## ページリロード時の挙動

- アクティブセッションはメモリ上にのみ存在する
- リロードするとセッションが消える（MVPではこれを許容する）
- DBに保存されるのは `ended` になった瞬間のみ

## フィールドとテーマの関係

- フィールド = ユーザーが定義した「用途」（例：英語、コーディング）
- テーマ = フィールドに紐づく「職業」（鉱夫・漁師・収集家）
- 1つのフィールドは1つのテーマを持つ
- テーマはアイテムのドロップ種別を決める
